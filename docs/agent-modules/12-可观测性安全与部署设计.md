# 可观测性、安全与部署设计

## 1. 模块定位

本模块覆盖三类横切能力：可观测性、安全治理和基础部署。它们不直接决定某一步怎么执行，但决定平台是否稳定、可审计、可扩展、可上线。

## 2. 可观测性设计

### 2.1 Metrics

建议至少监控：

- Run 数量
- 完成率
- 中断率
- 平均轮次
- 工具调用次数
- 模型调用耗时
- 工具耗时
- 审批等待时长
- 单 Run 成本

### 2.2 Logging

建议分层日志：

- API Log
- Runtime Log
- Model Log
- Tool Log
- Policy Log

### 2.3 Tracing

建议为以下对象分配 trace/span：

- AgentRun
- ModelCall
- ToolExecution
- Retrieval
- Approval
- Handoff

## 3. 安全与治理设计

### 3.1 鉴权

必须支持：

- 用户身份鉴权
- 服务身份鉴权
- 工具级权限控制

### 3.2 数据隔离

必须支持：

- Tenant 隔离
- User 隔离
- 环境隔离

### 3.3 Prompt 安全

建议支持：

- Prompt 模板版本管理
- Prompt 注入防护
- 输入输出审查

### 3.4 工具安全

建议支持：

- allowlist
- denylist
- 参数黑白名单
- 敏感工具审批

### 3.5 审计

关键动作必须可追踪：

- 谁发起了请求
- 调用了哪个 Agent
- 执行了哪些工具
- 谁审批了写操作

### 3.6 策略求值顺序

建议固定如下顺序：

1. 身份鉴权
2. 租户和环境隔离
3. Agent allowlist
4. Tool allowlist/denylist
5. 参数级校验
6. 审批策略
7. 执行

## 4. 基础设施建议

基础能力包括：

- DB
- Cache
- Queue
- Object Storage
- Vector Store
- Secret Manager

## 5. 部署形态建议

### 5.1 MVP 形态

单体服务即可：

- API
- Runtime
- Tool Executor
- DB
- Cache

### 5.2 平台化形态

建议拆分：

- API Service
- Runtime Service
- Tool Service
- Retrieval Service
- Approval Service
- Admin Service

### 5.3 异步化建议

- Run 创建同步返回
- 实际执行异步推进
- 前端通过事件流或轮询获取结果

## 6. 演进路径建议

### 6.1 Phase 1

- 单 Agent
- Tool Use
- 基本上下文
- 基本日志

### 6.2 Phase 2

- Router Agent
- 子 Agent handoff
- 审批流程
- 检索增强

### 6.3 Phase 3

- 运行恢复
- 摘要记忆
- 成本治理
- 完整可观测性

### 6.4 Phase 4

- 多 Agent 协同
- 智能策略调整
- 经验学习
- 自动化评测闭环

## 7. 风险控制点

- 没有可观测性就无法稳定运营长任务 Agent
- 没有隔离和审批就无法开放高权限工具
- 没有异步与恢复骨架就无法支撑真实业务负载
- 过早引入复杂多 Agent 会拖慢第一阶段落地

## 8. 验收重点

- 关键运行链路具备 Metrics、Logs、Tracing
- 关键动作具备身份、租户、工具三级控制
- MVP 到平台化拆分路径清晰
- 演进阶段与能力引入顺序符合源设计文档约束
